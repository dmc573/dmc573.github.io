2. Abstraction is when a complicated process is made simpler by covering up unnecessary details. For example, the iPhone uses abstraction to make it easier to use since there are less settings to worry about, but it loses functionality as a result. Abstractions in code are very useful, since it makes the code more readable and easier to troubleshoot.
3. One abstraction I used is the method their_history.count('b'). This is an abstraction of a loop that will cycle through the entire string their_history, but since I wanted to use it multiple times in my code, rather than coding the loop itself, I just used the method to make my code easier to read and edit.
